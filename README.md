# 📚 C++ 알고리즘 문제 풀이 환경

이 프로젝트는 여러 개의 C++ 소스 파일(`exam_code` 폴더)을 하나의 공통 환경(`main.cpp`)에서 효율적으로 빌드하고 테스트하기 위한 환경입니다.

---

## 📂 폴더 구조

.
├── main.cpp            # 공통 실행 로직 및 헤더 포함 (고정)
├── Makefile            # 빌드 자동화 스크립트
├── app.exe             # 생성된 실행 파일 (빌드 후 생성)
└── exam_code/          # 각 문제의 소스 코드가 위치하는 폴더
    ├── 2557.cpp
    ├── 1000.cpp
    └── ...

---

## 🛠️ 개발 환경 및 설정

* **Compiler**: `g++` (GCC)
* **Standard**: C++17
* **Encoding**: 
  * 소스 코드: `UTF-8`
  * 실행 환경: Windows CMD (`CP949`) 대응 설정 완료
* **Build Tool**: `make` (Windows용)

---

## 🚀 사용 방법

skeleton.cpp 파일에 명시된 대로 문제 풀이에 대한 내용은 각 코드 파일의 _main()함수에 구현하고, 터미널(CMD)에서 프로젝트 루트 폴더로 이동한 뒤, 문제 번호를 입력하여 빌드합니다.

### 1. 문제 빌드 (Build)
`make` 뒤에 실행하고자 하는 **문제 번호**를 입력하세요. (확장자 `.cpp`는 생략합니다.)

```bash
make 2557

해당 번호의 파일이 exam_code 폴더 내에 없을 경우 에러 메시지가 출력됩니다.

### 2. 실행 (RUN)
빌드가 완료된 후 아래 명령어로 프로그램을 실행합니다.

```bash
.\app.exe

### 3. 청소 (Clean)
생성된 실행 파일을 삭제하려면 다음 명령어를 입력합니다.

```bash
make clean


## ⚠️ 주의 사항
1. 파일명: exam_code 폴더 내의 파일 이름은 2557.cpp와 같이 숫자 또는 지정된 형식이어야 합니다.

2. 줄바꿈 및 인코딩: 윈도우 환경에 최적화되어 있으므로, 다른 OS(Linux/Mac)에서 사용 시 Makefile 내의 del 명령어를 rm으로 수정해야 합니다.
